在python中，函数的收集、参数列表的写法挺多的

>>> import sys
>>> def fun():
return 'No params'
>>> fun()
'No params'
#空参数列表,使用函数不可以传参
>>> 
>>> def fun_0(a,b,c):
return a,b,c

>>> fun_0('a','b',3)
('a', 'b', 3)
#一般的参数列表,需要对应好参数固定位置和数量
>>> 
>>> def fun_1(a,b,*c):
print a
print b
print c
>>> fun_1('a','b')
a
b
()
>>> fun_1('a','b','c')
a
b
('c',)
>>> fun_1('a','b','c','d','e')
a
b
('c', 'd', 'e')
#带有*的参数列表，*会把对应位置一般函数之后的所有参数容纳进来。*一定要放在一般参数的最后
>>> 
>>> def fun_2(a,b,c='c'):
print a
print b
print c
>>> fun_2('a','b')
a
b
c
>>> fun_2('a','b','cd')
a
b
cd
>>> fun_2('a','b',c='cd')
a
b
cd
>>> fun_2('a',c='cd','b')
SyntaxError: non-keyword arg after keyword arg
#带有"="的参数列表，同一般参数一样，要对应好位置，可以为空，但是要尽量放在一般参数列表后面，否则可能会引起异常。
>>> 
>>> def fun_3(a,b,c='c',*ds):
print a,b,c
print ds
>>> fun_3('a','b','c',1,2,4,5,6,7,'d')
a b c
(1, 2, 4, 5, 6, 7, 'd')
#带*的函数列表
>>> def fun_3(a,b,c='c',**ds):
print a,b,c
print ds
>>> fun_3('a','b','c',1,2,4,5,6,7,'d')

Traceback (most recent call last):
  File "", line 1, in
    fun_3('a','b','c',1,2,4,5,6,7,'d')
TypeError: fun_3() takes at most 3 arguments (10 given)
>>> 
>>> fun_3('a','b','c',d='d',m='m')
a b c
{'m': 'm', 'd': 'd'}
#带**的的函数列表，不接受直接传参，需要用赋值的形式
>>> def fun_4(a,b,c='c',*ds,**es):
print a,b,c
print ds
print es
>>> fun_4('a','b','c',1,2,4,5,6,7,'d',e='e',f='f')
a b c
(1, 2, 4, 5, 6, 7, 'd')
{'e': 'e', 'f': 'f'}
>>> 
将一般参数、"="参数、"*"参数、"**"参数整合到一块，可以发现，尽量要依次写他们的顺序。
>>> def fun_4(a,b,c='c',*ds,**es,*fs):
SyntaxError: invalid syntax
>>> 
由此可见，**参数是最后的，不能再添别的参数。
当然，类似的处理有系统自带sys.argv 命令行的处理
