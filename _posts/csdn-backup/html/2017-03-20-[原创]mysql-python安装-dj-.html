有一步操作，brew 这个是mac机器上的等同于apt-get ，不用理会。
在操作19：15的时候，easy_install mysql-python 出现的错误。
ImportError: Entry point ('console_scripts', 'easy_install') not found
看了一下 牛皮糖的Blog 在 《Ubuntu上修复Python》中提到是因为easy_install有两个路径，果不其然
which -a easy_install

然后我显现，在/usr/local/bin 其实还有一个easy_install-2.7
同样也会报错，对比命令的错误那一行“”，这就可以排除牛皮糖同学的说法，不是冲突的问题，

看2.7的这个内容：
---------------------------------------------
#!/usr/bin/python
# EASY-INSTALL-ENTRY-SCRIPT: 'distribute','console_scripts','easy_install-2.7'
__requires__ = 'distribute'
import sys
from pkg_resources import load_entry_point

if __name__ == '__main__':
    sys.exit(
        load_entry_point('distribute', 'console_scripts', 'easy_install-2.7')()
    )
------------------------------------------------
http://pythoner.org/wiki/830/ 一文中提到更新安装distribute_setup.py，同牛皮糖是一个想法，
根据作者的提示：
curl -O http://python-distribute.org/distribute_setup.py
sudo python distribute_setup.py
再运行easy_install mysql-python
报错：EnvironmentError: mysql_config not found
还是报错：这次语气好点，提示权限不足，再sudo试试看。
按照Alex Zhou 《python学习：安装mysql-mython》一文做法：
sudo apt-get install libmysqld-dev
再试试：sudo easy_install-2.7 mysql-python   # -----ok！

提示完成了。
然后find / -name mysql_config
也找到了mysql_config 。
